# Jellyfin Subtitles Tools (迅雷字幕内封插件)

`jellyfin-plugin-subtitles-tools` 是一款专为 Jellyfin 打造的现代化字幕管理插件，它与 [`subtitles-tools`](https://github.com/yexi-by/subtitles-tools) Python 服务配合使用，提供精准的迅雷字幕匹配，并采用**强制内封 (Soft-sub)** 的策略，带来极简、整洁的媒体库体验。

通过计算视频的 `CID/GCID`（迅雷文件哈希），插件能够实现最高精度的字幕匹配。同时，为了避免哈希值和外挂字幕文件的丢失与混乱，插件会自动将视频转换为 `MKV` 格式，将哈希值永久保存在 `MKV` 元数据中，并将下载的字幕直接内封进视频文件中。

> **插件仓库清单地址：**
> `https://raw.githubusercontent.com/yexi-by/jellyfin-plugin-subtitles-tools/main/manifest/stable.json`

## ✨ 核心特性

- **🎯 精准的迅雷字幕匹配：** 基于文件底层哈希 `CID/GCID` 搜索字幕，彻底解决重命名导致匹配失败的问题。
- **📦 极致整洁的内封策略：**
  - 自动将字幕转换为标准 `UTF-8 SRT` 格式。
  - 直接将字幕内封至 `MKV` 容器中，并在完成后自动清理临时文件，告别杂乱的外挂字幕。
- **🔄 自动入库与纳管：**
  - 新视频入库时，自动计算 `CID/GCID` 并转换为 `MKV` 格式，将哈希值写入文件元数据永久保存。
- **🖥️ 深度集成的 WebUI：**
  - 在电影/剧集详情页无缝注入「内封字幕」管理面板，告别繁琐的后台操作。
  - 支持多重分段视频（Multipart）的独立管理和一键批量处理。
- **🛠️ 完善的历史数据处理：** 提供 Jellyfin 计划任务（“一键算 CID/GCID 并转换为 MKV”），一键补全已有媒体库的数据。

## 📋 环境要求与支持范围

- **Jellyfin 版本:** 仅支持 `10.11.6` 版本。
- **媒体类型:** 仅支持本地可直接读取的 `Movie` (电影) 和 `Episode` (剧集)。
- **不支持:** `.strm` 等流媒体文件。
- **字幕类型:** 仅支持能稳定转换为文字型 `SRT` 的字幕，暂不支持图片字幕 (PGS/VOBSUB 等) 或 OCR 提取。
- **依赖服务:** 需要额外部署 [`subtitles-tools`](https://github.com/yexi-by/subtitles-tools) Python 核心服务。

## 🚀 安装与配置向导

### 第一步：部署核心服务
在安装此插件之前，您必须先部署 Python 核心后端。
请前往 [subtitles-tools 仓库](https://github.com/yexi-by/subtitles-tools) 按照说明启动服务。

### 第二步：安装 Jellyfin 插件
1. 进入 Jellyfin 控制台 -> **控制台 (Dashboard)** -> **插件 (Plugins)** -> **存储库 (Repositories)**。
2. 点击 **添加 (Add)**，输入以下信息：
   - **名称:** Subtitles Tools
   - **URL:** `https://raw.githubusercontent.com/yexi-by/jellyfin-plugin-subtitles-tools/main/manifest/stable.json`
3. 切换到 **目录 (Catalog)** 标签页，找到 `Subtitles Tools` 并安装。
4. 重启 Jellyfin 服务器以加载插件。

### 第三步：插件配置
进入 Jellyfin 插件页，点击 `Subtitles Tools` 进行配置：
- **Python 服务地址:** 填写第一步中部署的服务地址（例如：`http://127.0.0.1:8055`）。
- **FFmpeg 可执行文件路径（可选）:** 如果系统环境变量中没有配置 `ffmpeg`，请在此处指定完整路径（用于视频转换和字幕内封）。
- **新视频入库后自动纳管为 MKV:** 建议开启（默认开启）。新视频扫库且非 MKV 时会自动计算哈希并转为 MKV。已是 MKV 的文件会补算并写入。
- **视频转换并发数:** 建议根据 CPU 性能设置，默认为 `1`。
- **请求超时（秒）:** 默认即可。

> **提示：** 配置完成后，请点击下方的 **“测试连接”** 按钮，确认能正常连通 Python 服务及 FFmpeg 环境。

## 🎮 使用指南

### 1. 媒体详情页 WebUI (推荐入口)
安装并配置完成后，进入任意支持的电影或剧集详情页，您会看到一个新增的 **「内封字幕」** 按钮。
点击进入管理面板，您可以执行以下操作：
- **查看分段状态：** 查看当前媒体包含的所有视频分段、MKV 纳管状态、当前容器格式及已内封的字幕流。
- **手动纳管：** 手动将当前未纳管的分段或一键将整组分段转换为 `MKV`。
- **字幕搜索与内封：** 
  - 对单个当前分段搜索字幕候选，一键下载并内封。
  - 使用 **“最佳匹配并内封”** 一键为整组视频分段自动匹配最合适的字幕并完成内封。
- **清理维护：** 快速删除由插件写入的内封字幕流。

### 2. 计划任务 (处理历史媒体库)
针对安装插件前就已经存在于库中的媒体，Jellyfin 原生控制台的 **计划任务 (Scheduled Tasks)** 页面中会新增一个任务：
👉 **`一键算 CID/GCID 并转换为 MKV`**
手动运行此计划任务，即可批量将历史库中的视频归一化纳管为规范的 MKV 格式。

## ⚙️ 转换与内封规则说明

为了保证播放兼容性和文件一致性，本插件制定了严格的底层执行策略：
1. **目标容器唯一：** 所有处理后的视频统一强制封装为 `MKV` 格式。
2. **无损封装优先：** 格式转换时优先尝试 `remux` (仅改变容器，不重新编码音视频)。若 `remux` 失败，则自动回退到转码流程后再输出 `MKV`。
3. **字幕格式归一：** 下载的原始字幕会先经过统一转换，生成标准 `UTF-8 SRT` 编码的临时文件（如 `GIGP-123.srt`、`GIGP-54-cd1.srt`）。
4. **阅后即焚：** 字幕成功内封进 MKV 后，临时 `.srt` 文件会立即被清理，**插件不再保留任何外挂字幕文件**。
5. **内聚的元数据：** 插件不再依赖旧版的外部哈希归档文件，一切以当前视频自身的 MKV 元数据为准。并且在启动时，插件会自动清理数据目录里旧的归档、旧缓存和残留临时文件。

## ⚠️ 注意事项

1. **独立运行体系：** 本插件**不再依赖** Jellyfin 原生的字幕搜索提供器体系。因此，`Subtitles Tools` **不会**出现在 Jellyfin 常规的字幕提供器列表中。
2. **操作入口唯一：** 详情页的 **「内封字幕」** 页面是当前版本的唯一推荐入口。
