# Jellyfin Subtitles Tools

> 为 Jellyfin 提供迅雷字幕搜索、下载与自动内封的字幕管理插件。
> 单插件运行，无需外接后端。

---

## 它能做什么

- **搜索字幕** — 接入迅雷字幕源，基于文件指纹精准匹配，不会因为重命名而搜不到。
- **自动内封** — 下载后自动转为 UTF-8 SRT 并写入视频文件，播放时直接在播放器里选字幕。
- **自动转换** — 新视频入库时自动转为 MKV 格式，把文件指纹写进元数据，后续匹配一劳永逸。
- **批量处理** — 通过计划任务一键处理历史媒体库，也支持在详情页对多段视频整组操作。

---

## 安装

### 1. 添加插件仓库

在 Jellyfin 控制台中依次进入 **控制台 → 插件 → 存储库**，点击添加：

| 字段 | 值 |
|------|-----|
| 仓库名称 | `Subtitles Tools` |
| 仓库地址 | `https://raw.githubusercontent.com/yexi-by/jellyfin-plugin-subtitles-tools/main/manifest/stable.json` |

### 2. 安装插件

切换到 **目录** 标签页，找到 `Subtitles Tools` 并安装，然后重启 Jellyfin。

---

## 配置

安装完成后进入插件设置页，主要配置项如下：

### 必做

点击 **检测字幕源** 按钮，确认迅雷字幕源、FFmpeg 和视频兼容性都正常。

### 保存方式

选择默认的字幕保存方式：

| 方式 | 效果 |
|------|------|
| 写入视频 | 字幕直接内封到视频文件里，播放时切换字幕轨道即可 |
| 另存字幕 | 在同目录生成独立字幕文件，适合需要保留外挂字幕的场景 |

### 自动转换

- **新视频入库时自动转换**（建议开启） — 新视频加入媒体库时，自动计算文件指纹并转为 MKV。
- **自动转换路径黑名单** — 不希望自动处理的目录，按行填写即可。手动操作不受黑名单影响。

### 高级

其余选项（迅雷接口地址、请求超时、缓存时长、同时转换数、FFmpeg 路径、QSV 设备路径）保持默认即可，出问题时再调整。

## 使用

### 详情页操作（日常使用）

打开任意电影或剧集的详情页，右下角会出现 **字幕处理** 按钮，点击进入管理面板：

1. **搜索字幕** — 点击"搜索当前文件"，等待结果列表出现
2. **保存字幕** — 选一个字幕候选，点击保存（按当前保存方式写入）
3. **切换文件** — 如果是多段视频，左侧列表可以切换
4. **整组操作** — 多段视频时，可以一键整组转换或整组自动匹配字幕
5. **删除字幕** — 在"已有字幕"列表中，可以删除由插件添加的字幕

如果文件状态显示"待处理"或"建议先转换"，先点 **转换兼容格式** 处理完再搜索。

### 计划任务（处理历史视频）

安装插件前已有的视频，去 **控制台 → 计划任务**，运行 **"批量处理历史视频（转为 MKV 并修复安卓播放兼容）"**。

## 支持范围

| 项目 | 说明 |
|------|------|
| Jellyfin 版本 | 10.11.6 |
| 媒体类型 | Movie（电影）和 Episode（剧集），仅限本地文件 |
| 字幕类型 | 文字型字幕（可转为 SRT），不支持 PGS/VOBSUB 等图片字幕 |
| 外部依赖 | 需要系统中有可用的 FFmpeg / FFprobe |

---

## 工作原理

插件的工作流程可以概括为三步：

1. **指纹识别** — 计算视频文件的哈希值作为唯一标识，写入 MKV 元数据永久保存
2. **字幕匹配** — 用文件指纹到迅雷字幕源搜索，精度远高于文件名匹配
3. **字幕写入** — 下载字幕后统一转为 UTF-8 SRT，直接内封进视频，清理临时文件

转换视频时优先使用无损 remux（只换容器，不重编码），仅在需要修复安卓播放兼容时才用 QSV 重编码。

---

## 注意事项

- 本插件**独立运行**，不依赖 Jellyfin 自带的字幕提供器，也不会出现在常规字幕提供器列表中。
- 操作入口是详情页右下角的 **字幕处理** 按钮。
- 字幕成功写入后，下载产生的临时文件会自动清理，不会留下外挂字幕。
- 不支持 `.strm` 等流媒体文件。
