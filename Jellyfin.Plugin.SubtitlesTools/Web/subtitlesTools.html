Subtitles Tools

当前版本面向 Jellyfin 10.11.6 及更新的 10.11 系列，只处理本地 Movie 和 Episode 文件。
插件现在只认视频自身的 MKV 元数据来判断是否已纳管，并统一走“确保纳管 -> 迅雷搜字幕 -> 转 UTF-8 SRT -> 软字幕内封到 MKV”的流程。
历史影片可通过 Jellyfin 计划任务“一键算 CID/GCID 并转换为 MKV”统一处理；任务会扫描整个媒体库，但只处理未纳管影片。
Python 服务端项目： https://github.com/yexi-by/subtitles-tools
示例：http://127.0.0.1:8055
建议保持 10 秒，网络较慢时可适当调大。
默认开启。新入库文件若已是 MKV，则会自动补算 CID/GCID 并写入 MKV 元数据；若不是 MKV，则会先计算 CID/GCID，再转换为 MKV 并写入元数据。
默认 1。转换是重 I/O 操作，网络盘和机械盘不建议开太高。
留空时会优先自动探测 Jellyfin 自带 FFmpeg，再回退到系统 PATH。
